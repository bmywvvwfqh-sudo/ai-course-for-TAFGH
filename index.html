📋 萬用起手式提示詞
填空版範本 — 貼給 Claude / ChatGPT
我有個 Google Sheet，A欄是＿＿＿， B欄是＿＿＿，C欄是＿＿＿。 我想要＿＿（描述你想自動化的功能）＿＿， 請幫我寫 GAS（Google Apps Script）程式碼。
小提醒：欄位描述越具體，AI 產出越準確。不用擔心說錯，AI 會反問你細節。
💡 Bonus 進階提示詞
改顏色標記
如果到期日已過，幫我把那一列自動標成紅色底色；30 天內的標黃色。
修改通知天數
我想改成只有剩 30 天才寄信，超過 30 天的不要通知，怎麼修改？
加上已通知記錄
每次寄信後，在 E 欄自動寫入「已通知」和今天的日期，避免重複通知。
遇到紅字錯誤時
出現這個錯誤訊息：【把錯誤文字貼在這裡】，請幫我修正程式碼。
✨ 介面美化提示詞（UX / UI）
什麼時候用這些提示詞？

當 AI 幫你做出來的介面或自動化腳本「功能上沒問題，但用起來不夠直覺」時， 用這些提示詞要求 AI 把使用體驗做得更好。 不需要懂程式，直接貼給 AI 就行。

操作回饋 — 讓使用者知道「有在動」
按鈕按下去後，請加上視覺回饋：按鈕文字改成「處理中…」並暫時變灰色，完成後再改回來。 這樣使用者不會以為程式當掉了。
空白狀態 — 資料為空時的友善提示
當表格沒有資料或搜尋結果為空時，不要讓畫面空白。請顯示一個友善的中文提示， 例如「目前沒有資料，請先在表格填入資訊」，並加上一個小圖示。
確認機制 — 防止誤操作
在執行「刪除」或「寄信」等不可逆的動作之前，請先彈出確認視窗， 問使用者「確定要執行嗎？」，讓使用者可以取消，避免誤操作。
手機適配（RWD）
這個介面要能在手機上正常使用。按鈕要夠大（至少 44px 高）， 字體至少 14px，版面要自動依螢幕寬度調整，不要出現橫向滾動條。
友善錯誤訊息 — 讓使用者看懂
發生錯誤時，不要只顯示紅字英文代碼。請用繁體中文說明「發生什麼問題」， 以及「下一步應該怎麼做」，讓不懂技術的使用者也能自行處理。
整體介面風格指定
請用繁體中文設計這個介面。整體風格要簡潔、專業，主色用深藍色系， 按鈕要有圓角，字體大小至少 15px，所有互動元素都要有 hover 效果， 讓非技術背景的醫療人員也覺得好用。
💻 GAS 程式碼：證照過期警報器
Google Sheet 欄位對照
欄位內容範例
A 欄人員姓名王小明
B 欄證照名稱放射師執照
C 欄到期日2025/12/31
D 欄通知 Emailabc@hospital.com

第 1 列請設為標題列，資料從第 2 列開始填入。

程式碼（完整版，直接貼入 Apps Script）
function checkLicenseExpiry() { const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet(); const data = sheet.getDataRange().getValues(); const today = new Date(); const NOTIFY_DAYS = 180; // ← 想提前幾天通知就改這裡 for (let i = 1; i < data.length; i++) { const name = data[i][0]; // A欄：人員姓名 const license = data[i][1]; // B欄：證照名稱 const expiry = new Date(data[i][2]); // C欄：到期日 const email = data[i][3]; // D欄：通知 Email if (!email || !expiry || isNaN(expiry)) continue; const daysLeft = Math.floor((expiry - today) / (1000 * 60 * 60 * 24)); if (daysLeft > 0 && daysLeft <= NOTIFY_DAYS) { const subject = `【證照提醒】${name} 的 ${license} 還有 ${daysLeft} 天到期`; const body = [ `親愛的負責人，`, ``, `${name} 的「${license}」將於`, `${Utilities.formatDate(expiry, 'Asia/Taipei', 'yyyy/MM/dd')} 到期，`, `距今還有 ${daysLeft} 天，請提早安排更新。`, ``, `此為系統自動寄出，請勿直接回覆。`, ].join('\n'); GmailApp.sendEmail(email, subject, body); } } }
安裝步驟（5 分鐘完成）
1
開啟你的 Google Sheet，點上方選單 「擴充功能」 → 「Apps Script」
會開啟一個新分頁，那就是程式編輯器。
2
把編輯器裡的所有預設文字全選刪掉，貼上上方複製的程式碼， 按 ⌘+S（Mac）或 Ctrl+S（Windows）儲存
3
點上方的 「執行」（▶ Run） 按鈕
第一次會彈出授權視窗 → 點「進階」→「前往（不安全）」→「允許」，這是正常流程。
4
設定每天自動執行：點左側 時鐘圖示（觸發條件） → 右下角 「新增觸發條件」 → 函式選 checkLicenseExpiry → 活動類型選 「時間驅動」→「每天」 → 儲存
5
完成！ 之後每天 AI 會自動掃表格，快到期就自動寄信。
🚀 延伸挑戰（選做）
初級

把「證照過期警報器」的欄位改成你科室真正用的名稱，叫 AI 重新產一版程式碼，讓它變成「你的版本」。

中級

想想你每週最痛的手工任務，用萬用起手式提示詞描述給 AI，看它能不能幫你寫出自動化腳本。

進階

加上 UX 提示詞：把「180 天」改成「30 天」和「7 天」兩個門檻，並要求 AI 加上確認視窗與友善的中文錯誤訊息。

⚙️ 進階應用：前台登錄 + 後台管理
升級版 vs 基本版

基本版需要管理員手動填入資料。升級版讓每位同事自己用手機或電腦登錄， 管理員只需在試算表按一個按鈕——或等系統每月自動寄信。

前台 — 工作人員

📱 登錄介面

  • 掃 QR Code 開啟網頁
  • 填姓名、證照、到期日
  • 送出後自動進試算表
→
後台 — 管理者

📊 試算表選單

  • 查看所有登錄資料
  • 一鍵發送提醒信
  • 設定每月自動執行
試算表欄位（工作表名稱：「證照資料」）
欄位內容範例
A 欄姓名王小明
B 欄證照名稱放射師執照
C 欄到期日2025/12/31
D 欄Emailabc@hospital.com
E 欄登錄時間（系統自動填入）
前台 — doGet() ／ submitLicense()
工作人員開啟 Web App 網址，看到登錄表單（doGet）； 送出後資料自動寫入試算表（submitLicense）。
後台 — onOpen()
試算表開啟時自動建立「🔔 證照管理」選單， 讓管理者一鍵呼叫「立即發送」或「設定自動執行」。
後台 — checkLicenseExpiry()
邏輯與基本版相同，升級版固定讀「證照資料」工作表， 執行後彈出「共寄出 N 封提醒信」確認視窗。
後台 — setMonthlyTrigger()
設定每月 1 號 09:00 自動執行，先刪除舊觸發器避免重複寄信。
程式碼（後台核心，完整版含前台 HTML 請點複製）

部署步驟（比基本版多兩步）
1
貼上程式碼儲存後，點選上方選單 「部署」→「新增部署」
類型選「網頁應用程式」
2
執行身分：我（你的 Gmail）
存取權：所有人（同事不需登入就能填表）
⚠️ 每次修改程式碼後要記得「部署新版本」，否則同事看到的還是舊版。
3
複製部署後的 Web App 網址，發給同事（或做成 QR Code 貼在布告欄）
4
重新整理試算表，點上方選單 「🔔 證照管理」→「初始化表格欄位」
只需做一次，會自動建立「證照資料」工作表與標題列。
5
同事填完後：選單 → 「立即檢查並寄送提醒」
之後點 「設定每月 1 號自動執行」，就可以忘掉這件事了。
